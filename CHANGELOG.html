Added
  • Sincronización Bidireccional Neural: Implementada comunicación en tiempo real entre el Jules Panel y el standalone de Claude Chat mediante BroadcastChannel, permitiendo que las respuestas de Claude y los mensajes del usuario aparezcan instantáneamente en ambos interfaces sin recargar.
  • Renderizado de Actividad Jules Mejorado: El análisis de actividad de Jules ahora utiliza títulos descriptivos reales (ej: "Mensaje de Jules", "Sesión Fallida") en lugar de placeholders numerados opacos.

  • Persistencia de Contexto Neural por Sesión: El historial de conversación con Claude dentro del Jules Panel ahora se guarda de forma independiente para cada sesión de Jules mediante la clave hy_neural_context_${sessionId}, permitiendo mantener hilos de razonamiento distintos por tarea.
  • Sincronización Neural en Tiempo Real: Implementado BroadcastChannel para sincronizar instantáneamente el contexto neural entre múltiples pestañas abiertas del Jules Panel.
  • Visualización de Actividad Jules en Chat: Los fragmentos del historial de Jules enviados a Claude se renderizan ahora con un estilo visual propio y la etiqueta "JULES ACTIVITY" para mejorar la legibilidad del hilo.
  • Cross-Reference de Repositorios (Jules Panel): Implementada validación en paralelo entre GitHub API y Jules API. Los repositorios sin la App de Jules instalada se muestran deshabilitados con un icono de candado y tooltip informativo. Se añadió entrada manual para rutas personalizadas.
  • Modo de Automatización "AUTO_CREATE_PR": El Modo Automático ahora envía explícitamente automationMode: "AUTO_CREATE_PR" a la API de Jules para habilitar flujos de trabajo autónomos completos.
  • Asociación Forzada de Contexto: El Panel Jules ahora respeta y fuerza el Repositorio y la Rama indicados en el handoff de Claude, incluyendo una instrucción automática para crear la rama si no existe.
  • Kanban Card Archiving (COMPLETADO): Restaurado el botón de archivar (fa-box-archive) en las tarjetas de la columna COMPLETADO (estados OK/Closed) para permitir mover tareas al Cementerio directamente desde el Kanban.
  • Jules Activity Feed: Implementada una nueva pestaña de Actividad en el Neural Chat que muestra logs de ejecución, planes y comandos de Jules en tiempo real con estética Dracula.
  • Renderizado Dinámico del Equipo: La sección "The Dream Team" en la landing page ahora se genera dinámicamente mediante JavaScript, permitiendo el uso de archivos JSON fuera de la carpeta _data de Jekyll.
  • Global Card Archiving (Jules Panel): Upgraded the internal organization system to be global for all users. Archived states are now persisted in _data/jules_panel_state.json via GitHub commits, ensuring team-wide synchronization of the Kanban board view.
  • Internal Card Archiving (Jules Panel): Users can now drag session cards from EN COLA/EN PROGRESO to COMPLETADO or BLOQUEADO columns for internal organization. Archived cards are visually dimmed and display an "Archived" badge.
  • Relaunch Jules Session: Added a "Relanzar Jules" button to every Kanban card in Jules Panel. This button pre-fills the prompt, repository, and branch context for a new session based on the selected card.
  • Navegación Superior Jules Panel: Migrada la navegación principal a una barra superior en escritorio para optimizar el espacio de trabajo.
  • Sidebar Colapsable: Implementado sidebar retráctil (64px) con tooltips informativos y transiciones suaves.
  • Agrupación de Repositorios: Los repositorios en el sidebar ahora se agrupan por organización/dueño con headers colapsables.
  • Ghost Loading (Skeletons): Implementados placeholders animados (shimmer effect) para métricas y configuración mientras se cargan los datos.

  • Autenticación Neural Chat: Corregido el bug en claude-chat.html que bloqueaba la interfaz en el gate de "Acceso Restringido" tras una autenticación exitosa, asegurando una transición fluida al chat e implementando una inicialización idempotente.

  • Persistencia de Sesiones (Neural Chat): Corregida la lógica de guardado y carga de sesiones para evitar la pérdida de mensajes y asegurar la sincronización con el Jules Panel, incluyendo deduplicación de IDs.
  • Endpoints de Ollama (Error 404): Estandarizado el uso de /v1/chat/completions para todas las interacciones con Ollama, forzando compatibilidad con el estándar de OpenAI y eliminando fallos por rutas incorrectas.

  • SyntaxError en Jules Panel (V2): Corregido el error SyntaxError: Identifier 'archive' has already been declared que impedía la carga del Kanban de tareas. Se renombró la variable local duplicada a localArchive y se actualizaron todas sus referencias en la función loadJulesKanban.
  • Estabilidad del Jules Panel (v2): Implementado manejo de errores no bloqueante en la carga de datos globales. El fallo en la recuperación del archivo de estado o de la API de GitHub ya no detiene la inicialización del panel, garantizando que los menús y métricas básicas sigan siendo accesibles.

  • Estabilidad de AuthManager: Corregido el error TypeError: Cannot read properties of null (reading 'on') en auth.js mediante la implementación de guardas defensivas y el uso explícito de window.jQuery para evitar conflictos de shadowing con el helper $ en vistas como Jules Panel.
  • Estabilidad de Ejecución (Jules Panel): Implementada una guarda en la interfaz que deshabilita el botón "Iniciar Ejecución" mientras se cargan los repositorios, evitando el error FAILED_PRECONDITION causado por enviar placeholders vacíos.
