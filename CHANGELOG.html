Changelog de Hypenosys 🚀

Aquí listamos los avances del estudio. ¡Mantened esto actualizado!

Añadir Nueva Entrada

[Unreleased]

Added
  • Sincronización de Documentación: Nueva sección de Documentación que lee en tiempo real el repositorio público hypenosys/docs.
    • Arquitectura Live-Sync: Integración directa con la API de GitHub para cargar y renderizar Markdown en tiempo de ejecución, eliminando la necesidad de copias manuales.
    • Sidebar Navegable: Menú lateral con soporte para directorios recursivos y búsqueda de documentos.
    • Integración con Obsidian: Botón dedicado para abrir notas directamente en la bóveda local de Obsidian vía URI protocol.
    • Estética Dracula/Obsidian: Diseño oscuro coherente con el resto del ecosistema Hypenosys, optimizado para lectura técnica.
    • Quick Jump: Acceso rápido desde la landing page y el header principal.
  • Motor de Estadísticas Neurales v1.1.0: Rediseño total de la arquitectura de métricas del estudio.
    • Layer A (Individual Performance): Nuevo modal "Deep Dive" para cada miembro con análisis de contribución por tipo de tarea y rama, velocidad de Story Points semanal (Chart.js), impacto en dependencias, métricas de colaboración (comentarios) y badges de especialización automáticos ("Tag Intelligence").
    • Layer B (Studio Group Stats): Nueva sección central con indicadores de Bug Rate, Blocker Health, Stress Index y tareas restantes para el ship. Incluye tendencia de velocidad histórica y mapa de calor de etiquetas.
    • Gamified Hall of Fame: Leaderboard renovado con categorías dinámicas (MVP, Bug Slayer, Unblocker, Velocity King, Researcher, Art Lead, Collaborator) con reseteo y archivado automático por Milestone.
    • Cómputo Histórico: El sistema ahora procesa tanto tareas activas como archivadas (Cementerio) para generar perfiles de carrera completos.
  • Sistema de Tareas v1.1.0: Evolución completa del schema de tareas para soportar gestión avanzada de proyectos.
    • Nuevos Campos: Título, Fechas de inicio/vencimiento, Horas estimadas, Story Points, Tipo de tarea (Feature/Bug/Chore/Research) y Criterios de Aceptación con Markdown.
    • Gestión de Listas: UI interactiva para Tags (chips), Enlaces Externos y Subtareas con estado de completitud.
    • Historial Automático: Sistema de change_log que registra cambios campo por campo en cada guardado, atribuyéndolos al usuario de GitHub autenticado.
    • Dependencias Mejoradas: Los campos "Bloquea" y "Bloqueada por" ahora resuelven automáticamente los IDs a títulos de tareas para mayor claridad.
    • Migración Automática: Lógica de actualización transparente de v1.0.0 a v1.1.0, incluyendo extracción inteligente de dependencias desde comentarios antiguos y generación de títulos.
  • Sistema de Persistencia de Sesión: Implementada la opción "Remember me" utilizando localStorage para sesiones persistentes y sessionStorage para temporales.
  • Auto-Login Mejorado: Corregido bug de persistencia que obligaba a re-autenticar tras refrescar la página. Ahora el sistema valida automáticamente el token guardado al cargar.
  • Soporte para login automático al cargar la aplicación si el token sigue siendo válido.
  • Protección contra bucles infinitos de redirección durante el flujo de autenticación.
  • Funcionalidad de Archivo y Cementerio de Sesiones en el Jules Panel y Dashboard de Hypenosys, permitiendo la gestión local de sesiones inconcluyentes y en pausa.
  • Sincronización de sesiones de Jules con el Dashboard de Operaciones para monitoreo en tiempo real desde una interfaz centralizada ("Godmode").
  • [DOCS] Añadida guía de configuración de Antigravity IDE y sincronización de Obsidian con el repositorio de documentación en guia-comandos.html, incluyendo navegación mejorada (ToC y botón de subir).
  • TortoiseSVN onboarding section and collaborative workflow guide in guia-ue5-svn.html
  • plan_de_arranque.html unified with platform design system
  • CHANGELOG enforcement GitHub Action
  • Botón de toggle inteligente en el Kanban de Operaciones para expandir o contraer todas las tareas de forma global, con persistencia en el navegador.
  • Alias de Repositorios en Jules Panel: Implementado sistema para asignar nombres personalizados (alias) a repositorios. Los alias son locales por usuario (localStorage) y no afectan a los datos reales de GitHub, facilitando la identificación de proyectos. Incluye selector con lápiz de edición, modal de configuración y persistencia tras recarga.
  • Integración de Imágenes via Tumblr: Implementado sistema de hosting externo para imágenes utilizando un Cloudflare Worker y la API de Tumblr. Esto reduce el tamaño del repositorio al evitar el almacenamiento de binarios Base64.
    • Nuevo input dual en el modal de tareas: Adjuntar archivo (subida a Tumblr) o Pegar URL.
    • Migración automática del schema anterior a un formato unificado basado en URLs.
    • Badge ⚠️ legacy para identificar imágenes antiguas almacenadas en el repo.
    • Lógica de limpieza al archivar: el sistema ofrece eliminar físicamente las imágenes legacy del repositorio para liberar espacio.
    • Límite de subida ampliado a 10MB.
  • Soporte de Imágenes en Kanban: Implementado soporte para adjuntar imágenes a las tareas. Incluye carga mediante selección de archivos, arrastrar y soltar, y pegado directo desde el portapapeles (Ctrl+V). Las imágenes se visualizan en galerías colapsables dentro de las tarjetas del Kanban y en un visor Lightbox a pantalla completa.
Fixed
  • Lightbox — Rewrite de Interacción Táctil (Critical Fix): Corregido definitivamente el problema de "eventos fantasma" en dispositivos móviles donde los thumbnails requerían dos toques para abrirse o encolaban el evento para la siguiente interacción.
    • Eliminado el uso de etiquetas <button> y listeners inline en el Kanban para evitar conflictos de foco y competencia con el sistema de drag-and-drop.
    • Implementada inyección vía DOM de thumbnails usando <div role="button"> con tabindex="-1" para eliminar el anillo de foco y el retardo del navegador.
    • Uso de pointerdown con captura agresiva (capture: true), preventDefault() y stopImmediatePropagation() para garantizar apertura inmediata.
    • Implementado bypass temporal de draggability del card padre y fallback de touchstart para máxima compatibilidad en tablets.
    • Limpieza de foco automática mediante document.activeElement?.blur() al cerrar el Lightbox.
    • Eliminación global de anillos de foco y highlight táctil en elementos con role="button".
  • Bug Follow-up — Retardo en apertura de Lightbox: Corregido definitivamente el problema donde las fotos de las tareas requerían dos interacciones para abrirse en dispositivos táctiles.
    • Implementado el uso de onpointerdown con event.preventDefault() para interceptar la interacción antes de la separación de eventos de foco/click del navegador.
    • Migración de contenedores de imágenes a etiquetas semánticas <button> para mejorar la prioridad de eventos y la accesibilidad.
    • Añadido soporte para teclado (Enter/Espacio) mediante onkeydown para mantener la accesibilidad universal.
    • Uso de focus-visible para suprimir el anillo de foco en interacciones de puntero mientras se preserva para usuarios de teclado.
  • Retardo en apertura de Lightbox: Corregido bug donde las fotos de las tareas no se abrían inmediatamente en dispositivos táctiles/tablets. Se implementó event.stopPropagation() y draggable="false" para evitar conflictos con el sistema de arrastre del Kanban.
  • Navegación en Lightbox: Añadidas flechas de navegación y contador de imágenes para permitir ciclar entre múltiples fotos de una tarea sin cerrar el visor.
  • Soporte de Teclado: Implementado cierre con Escape y navegación con flechas de dirección en el visor de imágenes.
  • Error de Sincronización y Paneles Vacíos: Corregido crash fatal "Cannot read properties of undefined (reading 'all_time')" y optimizado el motor de estadísticas para evitar paneles vacíos tras la migración v1.1.0. Se implementaron fallbacks inteligentes (agrupación por creación si no hay completitud, etiquetas sintéticas desde tipo/rama) y se recalibraron las métricas grupales para incluir tareas activas.
  • Regresión en Botón de Edición: Corregido error que impedía abrir el modal de edición de tareas debido a comprobaciones de tipo estrictas (===) entre IDs numéricos y cadenas provenientes del DOM.
  • Responsive layout and horizontal overflow issues on mobile (320px-430px) for Dashboard and Jules Panel.
  • Markdown rendering in progressUpdated and sessionCompleted activities in Jules Panel.
  • Removed duplicate "Jules Panel" title rendered by the Jekyll layout.
  • Applied active state and disabled link for "Jules" in the navbar when on the Jules Panel page.
  • Active state highlighting for "Comandos" and "Tech Stack" links in the main navigation.
Removed
  • "JULES" navigation item from the dashboard menu to eliminate duplication with "Quick Jules" button.
