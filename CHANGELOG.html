Changelog de Hypenosys 🚀

Aquí listamos los avances del estudio. ¡Mantened esto actualizado!

Añadir Nueva Entrada

[Unreleased]

Fixed
  • OAuth Race Condition Fix: Implementada arquitectura cooperativa entre AuthManager y dashboard-data.js.
    • Modo Cooperativo: AuthManager ahora expone una promesa isReady y cede el control del intercambio del token OAuth al dashboard cuando se detecta un código en la URL.
    • Limpieza de URL: Se ha integrado history.replaceState para eliminar parámetros sensibles (?code=) inmediatamente después de la autenticación.
    • Gestión de Estado: Uso de sessionStorage.oauth_in_progress para evitar que los guards de autenticación interfieran durante el proceso de login.
Added
  • Sistema Unificado de Perfiles: Migración total a team_profiles.json como única fuente de verdad, eliminando la dependencia de team.json.
    • Nuevo Schema v2.0: Implementados campos de lead_role, is_admin, skills, extra_links dinámicos, y redes sociales (Twitter/Itch).
    • Modal Compartido: Nuevo partial _includes/profile_modal.html que unifica la edición de perfiles con soporte para todos los nuevos campos y sincronización de color de acento.
    • Modo Administrador: Usuarios con is_admin: true pueden editar cualquier perfil del equipo mediante un selector integrado en el modal.
    • Runtime Fetch: La landing page ahora carga los datos del equipo dinámicamente vía API de GitHub, garantizando que los cambios sean visibles al instante sin redespliegues.
    • Fix en Ops Dashboard: El avatar del dashboard ahora es un dropdown funcional con acceso a "Mi Perfil", "Ajustes" y "Logout", consistente con la landing page.
    • Escritura Atómica: Refactorizado github-api.js para manejar actualizaciones complejas del perfil con resolución de conflictos 409.
  • Aliases de Repositorios en Dashboard: Implementada la visualización de aliases personalizados en el modal de tareas y en el footer de las Kanban cards.
Changed
  • UI de Kanban Cards: Reordenados los elementos de la cabecera de las tarjetas para mejorar el flujo de lectura. El botón de edición se ha movido a la izquierda, junto al ID de la tarea, mientras que las acciones de archivo y colapso se mantienen a la derecha.
  • Estrategia de Carga Híbrida en Jules Panel: Optimización radical del selector de repositorios para eliminar tiempos de espera y timeouts.
    • Phase 1 (GitHub API): Carga inmediata de repositorios de la organización Hypenosys utilizando caché local o GitHub API directa. Los repositorios están disponibles para selección instantánea con etiquetas de nombre corto.
    • Phase 2 (Jules API Enrichment): Enriquecimiento en segundo plano de las etiquetas con alias oficiales de Jules y adición de repositorios de otras organizaciones, con un timeout de seguridad de 8 segundos.
    • Gestión de Estado de Sesión: El botón "Lanzar Sesión" ahora muestra un estado de verificación visual y se habilita automáticamente tras la validación con la API de Jules, garantizando compatibilidad total.
    • Persistencia Inteligente: Se preserva la selección del usuario y la rama elegida durante el proceso de enriquecimiento, evitando reseteos de formulario.
  • Filtro de Miembros Optimizado: Rediseño del selector de equipo en el Dashboard para mejorar la usabilidad en dispositivos móviles y tablets.
    • Layout de Chips Horizontales: Reemplazada la lista vertical por una fila única de chips/pills compactos con scroll horizontal suave.
    • Indicador de Scroll Visual: Añadido un gradiente de desvanecimiento (fade) en el borde derecho para indicar intuitivamente la existencia de más miembros.
    • Diseño Responsivo: Implementadas clases dinámicas que ajustan el padding y tamaño de fuente entre mobile (xs) y desktop (sm).
    • Consistencia Visual: El diseño de chips ahora es uniforme en todas las resoluciones, manteniendo el esquema de colores Dracula y el resaltado del filtro activo con el color accent verde.
    • Reset Inteligente: El chip "Todos" ahora siempre aparece en primera posición y resetea todos los filtros (miembros y etapas) al pulsarse.
    • Layout Híbrido de Header: Implementado sistema de dos filas para móvil/tablet (Fila 1: Branding y Acciones, Fila 2: Chips) y fila única consolidada para desktop, garantizando visibilidad total sin desbordamientos.
    • Optimización de Espacio: Ocultado el timestamp de sincronización en móvil y simplificado el botón "Quick Jules" a solo icono para maximizar el área de trabajo en pantallas pequeñas.
Added
  • Visualización de Repositorio en Kanban: Implementada la visualización del repositorio asignado en las tarjetas del Kanban.
    • Integración en Cards: El repositorio (o su alias) se muestra centrado en el footer de la card, entre las flechas de navegación, tanto en modo expandido como minimizado.
    • Selector Dinámico en Modal: Añadido campo "Repositorio" en la pestaña Info del modal de tareas, que se popula automáticamente con los repositorios de la organización Hypenosys vía GitHub API.
    • Caché de Rendimiento: Los repositorios se cargan en background una sola vez al inicializar el dashboard para garantizar una interfaz fluida.
    • Soporte de Alias: Integración con el sistema de alias existente para mostrar nombres amigables en lugar del nombre técnico del repo.
  • Sincronización de Documentación: Nueva sección de Documentación que lee en tiempo real el repositorio público hypenosys/docs.
    • Arquitectura Live-Sync: Integración directa con la API de GitHub para cargar y renderizar Markdown en tiempo de ejecución, eliminando la necesidad de copias manuales.
    • Sidebar Navegable: Menú lateral con soporte para directorios recursivos y búsqueda de documentos.
    • Integración con Obsidian: Botón dedicado para abrir notas directamente en la bóveda local de Obsidian vía URI protocol.
    • Estética Dracula/Obsidian: Diseño oscuro coherente con el resto del ecosistema Hypenosys, optimizado para lectura técnica.
    • Quick Jump: Acceso rápido desde la landing page y el header principal.
  • Motor de Estadísticas Neurales v1.1.0: Rediseño total de la arquitectura de métricas del estudio.
    • Layer A (Individual Performance): Nuevo modal "Deep Dive" para cada miembro con análisis de contribución por tipo de tarea y rama, velocidad de Story Points semanal (Chart.js), impacto en dependencias, métricas de colaboración (comentarios) y badges de especialización automáticos ("Tag Intelligence").
    • Layer B (Studio Group Stats): Nueva sección central con indicadores de Bug Rate, Blocker Health, Stress Index y tareas restantes para el ship. Incluye tendencia de velocidad histórica y mapa de calor de etiquetas.
    • Gamified Hall of Fame: Leaderboard renovado con categorías dinámicas (MVP, Bug Slayer, Unblocker, Velocity King, Researcher, Art Lead, Collaborator) con reseteo y archivado automático por Milestone.
    • Cómputo Histórico: El sistema ahora procesa tanto tareas activas como archivadas (Cementerio) para generar perfiles de carrera completos.
  • Sistema de Tareas v1.1.0: Evolución completa del schema de tareas para soportar gestión avanzada de proyectos.
    • Nuevos Campos: Título, Fechas de inicio/vencimiento, Horas estimadas, Story Points, Tipo de tarea (Feature/Bug/Chore/Research) y Criterios de Aceptación con Markdown.
    • Gestión de Listas: UI interactiva para Tags (chips), Enlaces Externos y Subtareas con estado de completitud.
    • Historial Automático: Sistema de change_log que registra cambios campo por campo en cada guardado, atribuyéndolos al usuario de GitHub autenticado.
    • Dependencias Mejoradas: Los campos "Bloquea" y "Bloqueada por" ahora resuelven automáticamente los IDs a títulos de tareas para mayor claridad.
    • Migración Automática: Lógica de actualización transparente de v1.0.0 a v1.1.0, incluyendo extracción inteligente de dependencias desde comentarios antiguos y generación de títulos.
  • Sistema de Persistencia de Sesión: Implementada la opción "Remember me" utilizando localStorage para sesiones persistentes y sessionStorage para temporales.
  • Auto-Login Mejorado: Corregido bug de persistencia que obligaba a re-autenticar tras refrescar la página. Ahora el sistema valida automáticamente el token guardado al cargar.
  • Soporte para login automático al cargar la aplicación si el token sigue siendo válido.
  • Protección contra bucles infinitos de redirección durante el flujo de autenticación.
  • Funcionalidad de Archivo y Cementerio de Sesiones en el Jules Panel y Dashboard de Hypenosys, permitiendo la gestión local de sesiones inconcluyentes y en pausa.
  • Sincronización de sesiones de Jules con el Dashboard de Operaciones para monitoreo en tiempo real desde una interfaz centralizada ("Godmode").
  • [DOCS] Añadida guía de configuración de Antigravity IDE y sincronización de Obsidian con el repositorio de documentación en guia-comandos.html, incluyendo navegación mejorada (ToC y botón de subir).
  • TortoiseSVN onboarding section and collaborative workflow guide in guia-ue5-svn.html
  • plan_de_arranque.html unified with platform design system
  • CHANGELOG enforcement GitHub Action
  • Botón de toggle inteligente en el Kanban de Operaciones para expandir o contraer todas las tareas de forma global, con persistencia en el navegador.
  • Alias de Repositorios en Jules Panel: Implementado sistema para asignar nombres personalizados (alias) a repositorios. Los alias son locales por usuario (localStorage) y no afectan a los datos reales de GitHub, facilitando la identificación de proyectos. Incluye selector con lápiz de edición, modal de configuración y persistencia tras recarga.
  • Integración de Imágenes via Tumblr: Implementado sistema de hosting externo para imágenes utilizando un Cloudflare Worker y la API de Tumblr. Esto reduce el tamaño del repositorio al evitar el almacenamiento de binarios Base64.
    • Nuevo input dual en el modal de tareas: Adjuntar archivo (subida a Tumblr) o Pegar URL.
    • Migración automática del schema anterior a un formato unificado basado en URLs.
    • Badge ⚠️ legacy para identificar imágenes antiguas almacenadas en el repo.
    • Lógica de limpieza al archivar: el sistema ofrece eliminar físicamente las imágenes legacy del repositorio para liberar espacio.
    • Límite de subida ampliado a 10MB.
  • Soporte de Imágenes en Kanban: Implementado soporte para adjuntar imágenes a las tareas. Incluye carga mediante selección de archivos, arrastrar y soltar, y pegado directo desde el portapapeles (Ctrl+V). Las imágenes se visualizan en galerías colapsables dentro de las tarjetas del Kanban y en un visor Lightbox a pantalla completa.
  • Prácticas Mobile-First Transversales: Rediseño global de la interfaz para máxima accesibilidad y UX en dispositivos móviles y tablets.
    • Visibilidad de Acciones WCAG: Introducción de la clase .action-btn con touch targets de 44x44px. Los botones críticos ahora son siempre visibles en dispositivos táctiles (detección via hover: none) y mantienen una visibilidad base del 45-70% en desktop, eliminando el patrón de "oculto hasta hover".
    • Kanban Mobile Layout: Reestructuración de las tarjetas en un layout de 5 filas (Meta, Título, Datos Clave, Equipo y Acciones). Implementados botones de transición de estado ("Anterior/Siguiente") para mobile, eliminando la dependencia del drag-and-drop.
    • Sistema de Tabs en Modal: El modal de tareas ahora cuenta con un sistema de 3 pestañas (Info, Equipo & Fechas, Extra) para evitar el scroll infinito en pantallas pequeñas, con reseteo automático al abrir.
    • Overlays de Imagen Responsivos: Las acciones de ver/borrar imágenes ahora se muestran en una barra de botones fija debajo de la imagen en mobile, preservando el overlay solo en desktop.
  • Gestión de Imágenes en Tareas: Restauración completa de la sección de imágenes en la pestaña "Extra" del modal de tareas.
    • Grid de Miniaturas: Implementado layout responsivo de 80x80px para previsualizar imágenes adjuntas.
    • Acciones Directas: Botones de visualización (Lightbox) y borrado siempre visibles en mobile y por hover en desktop.
    • Flujo de Subida: Integración del botón "+" para subidas directas a través del worker de Tumblr y restauración del soporte Drag & Drop.
    • Refresco Automático: La galería se actualiza dinámicamente al cambiar a la pestaña "Extra" o realizar modificaciones.
Fixed
  • Auth Hang en Jules Panel: Corregido error de sintaxis en jules-panel.html que bloqueaba la ejecución del script de autenticación y dejaba el panel atascado en "Verificando credenciales...".
  • Duplicación de Iconos en Kanban: Eliminada la duplicación de botones de acción (editar, archivar, colapsar) mediante la unificación de bloques responsivos.
    • Migración de clases custom .desktop-only/.mobile-only a clases nativas de Tailwind CSS (md:hidden, hidden md:flex).
    • Reubicación del botón de archivar a la parte superior derecha junto a las acciones principales.
    • Optimización del layout móvil: flechas de transición de estado ahora posicionadas en los extremos laterales inferiores para mejor ergonomía táctil (justify-between).
  • Layout de Tabs en Modal: Corregido error que renderizaba los tabs en vertical; ahora se fuerzan a una fila horizontal (flex-row) en todas las resoluciones.
  • Lightbox — Nuclear Minimalist Fix: Rediseño total de la interacción de imágenes en el Kanban para eliminar fricción y "clics fantasma".
    • Eliminado el botón colapsable "{n} imagenes" en favor de una previsualización directa y minimalista en la tarjeta.
    • Uso de div con touch-action: none para las miniaturas, forzando una respuesta instantánea al pointerdown.
    • Supresión agresiva de eventos concurrentes (touchstart, click) mediante stopImmediatePropagation para aislar completamente la apertura del visor del sistema de arrastre del Kanban.
    • Eliminado el retardo de seguridad (guard) de 300ms en el Lightbox para una respuesta inmediata.
  • Kanban — Solución Definitiva para Miniaturas (Nuclear Fix): Reemplazo completo del sistema de eventos táctiles por navegación nativa mediante etiquetas <a>.
    • Eliminados todos los listeners de pointerdown, touchstart y pointercancel que causaban conflictos en tablets.
    • Uso de onclick directo en el elemento anchor para máxima prioridad en el navegador.
    • Optimizado el listener de dragstart del card padre para no interferir con los clics en las miniaturas, manteniendo la funcionalidad de arrastre del card desde el fondo.
  • Lightbox — Pointer Events & Ghost Click Suppression: Reemplazo completo de click por pointerdown con stopImmediatePropagation en miniaturas del Kanban para una respuesta instantánea en tablets. Se ha añadido un listener de captura para anular "ghost clicks" y una breve suspensión del draggable del card (300ms) para evitar colisiones con el sistema de arrastre.
  • Kanban — Image Click & Drag Interference (Critical Fix): Solucionado el problema donde las miniaturas de imagen no abrían el visor de forma consistente debido a interferencias con el arrastre de la tarjeta y el uso de botones invisibles.
    • Eliminados los botones invisibles sobre las miniaturas en favor de listeners directos.
    • Implementado data-no-drag="true" y filtrado de dragstart para evitar el arrastre accidental al interactuar con imágenes, selectores o botones.
    • Estandarizada la lógica de previsualización de imágenes tanto en el Kanban como en el Modal de Edición.
    • Cambiado el estado por defecto de las tarjetas a minimizadas para una mejor visualización del tablero.
  • Dashboard — Corregida regresión de clicks en tarjetas: Solucionado el problema de "clicks fantasma" donde pulsar el botón de edición o elementos internos podía disparar accidentalmente el visor de imágenes o interferir con la apertura de modales.
    • Se ha implementado event.stopPropagation() en todos los elementos interactivos de las tarjetas (botones, selectores, avatares).
    • Refactorizada la lógica del Lightbox para usar un handler dedicado openImagePreview, eliminando los listeners globales a nivel de tarjeta que causaban interferencias.
    • Mejorada la responsividad y el z-index del Lightbox para garantizar una visualización correcta en dispositivos móviles.
  • Lightbox — Kanban Thumbnail Mobile & Tablet Fix: Corregido el problema donde las miniaturas de imagen en las tarjetas del Kanban no abrían el visor en dispositivos móviles y tablets al entrar en conflicto con el sistema de arrastre (draggable). Se implementó un sistema de delegación de eventos en el card padre que intercepta tanto touchstart como click sobre botones de miniatura usando capture: true, garantizando una respuesta inmediata y compatibilidad total entre dispositivos táctiles y de escritorio.
  • Lightbox — Rewrite de Interacción Táctil (Critical Fix): Corregido definitivamente el problema de "eventos fantasma" en dispositivos móviles donde los thumbnails requerían dos toques para abrirse o encolaban el evento para la siguiente interacción.
    • Eliminado el uso de etiquetas <button> y listeners inline en el Kanban para evitar conflictos de foco y competencia con el sistema de drag-and-drop.
    • Implementada inyección vía DOM de thumbnails usando <div role="button"> con tabindex="-1" para eliminar el anillo de foco y el retardo del navegador.
    • Uso de pointerdown con captura agresiva (capture: true), preventDefault() y stopImmediatePropagation() para garantizar apertura inmediata.
    • Implementado bypass temporal de draggability del card padre y fallback de touchstart para máxima compatibilidad en tablets.
    • Limpieza de foco automática mediante document.activeElement?.blur() al cerrar el Lightbox.
    • Eliminación global de anillos de foco y highlight táctil en elementos con role="button".
  • Bug Follow-up — Retardo en apertura de Lightbox: Corregido definitivamente el problema donde las fotos de las tareas requerían dos interacciones para abrirse en dispositivos táctiles.
    • Implementado el uso de onpointerdown con event.preventDefault() para interceptar la interacción antes de la separación de eventos de foco/click del navegador.
    • Migración de contenedores de imágenes a etiquetas semánticas <button> para mejorar la prioridad de eventos y la accesibilidad.
    • Añadido soporte para teclado (Enter/Espacio) mediante onkeydown para mantener la accesibilidad universal.
    • Uso de focus-visible para suprimir el anillo de foco en interacciones de puntero mientras se preserva para usuarios de teclado.
  • Retardo en apertura de Lightbox: Corregido bug donde las fotos de las tareas no se abrían inmediatamente en dispositivos táctiles/tablets. Se implementó event.stopPropagation() y draggable="false" para evitar conflictos con el sistema de arrastre del Kanban.
  • Navegación en Lightbox: Añadidas flechas de navegación y contador de imágenes para permitir ciclar entre múltiples fotos de una tarea sin cerrar el visor.
  • Soporte de Teclado: Implementado cierre con Escape y navegación con flechas de dirección en el visor de imágenes.
  • Error de Sincronización y Paneles Vacíos: Corregido crash fatal "Cannot read properties of undefined (reading 'all_time')" y optimizado el motor de estadísticas para evitar paneles vacíos tras la migración v1.1.0. Se implementaron fallbacks inteligentes (agrupación por creación si no hay completitud, etiquetas sintéticas desde tipo/rama) y se recalibraron las métricas grupales para incluir tareas activas.
  • Regresión en Botón de Edición: Corregido error que impedía abrir el modal de edición de tareas debido a comprobaciones de tipo estrictas (===) entre IDs numéricos y cadenas provenientes del DOM.
  • Responsive layout and horizontal overflow issues on mobile (320px-430px) for Dashboard and Jules Panel.
  • Markdown rendering in progressUpdated and sessionCompleted activities in Jules Panel.
  • Removed duplicate "Jules Panel" title rendered by the Jekyll layout.
  • Applied active state and disabled link for "Jules" in the navbar when on the Jules Panel page.
  • Active state highlighting for "Comandos" and "Tech Stack" links in the main navigation.
Removed
  • "JULES" navigation item from the dashboard menu to eliminate duplication with "Quick Jules" button.
