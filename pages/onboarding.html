Hypenosys · Pipeline Dev

Onboarding para nuevos devs del pipeline Hypenosys

Guía rápida para conectar GitHub, Jules y el flujo interno de Hypenosys antes de empezar a trabajar en tareas reales.

Para nuevos desarrolladores y colaboradores técnicos

Antes de empezar

  • Cuenta de GitHub — necesaria para todo el flujo.
  • Debes aceptar la invitación a la organización hypenosys en GitHub.
  • Debes tener acceso al repositorio de prueba Minecraft-Server-Docker.
  • El repositorio puede estar vacío inicialmente — es normal.
  • La primera tarea sirve para comprobar que Jules puede generar cambios y abrir una PR correctamente.
  • No uses un repositorio crítico para esta primera prueba.
Flujo oficial · jules.google.com
01Org

Aceptar la invitación a la organización Hypenosys

GitHub

Revisa tu correo o tus notificaciones de GitHub y acepta la invitación para entrar en la organización hypenosys. Comprueba que puedes ver la organización desde tu cuenta.

  • He aceptado la invitación a hypenosys
  • Veo la organización en mi lista de GitHub
  • Veo el repositorio Minecraft-Server-Docker
  • Si no lo veo, aviso al administrador del proyecto
02Jules

Crear cuenta o entrar en Jules

JulesGitHub

Entra en Jules e inicia sesión con tu cuenta de Google, creando la cuenta si es necesario. Vincula GitHub cuando Jules lo solicite y autoriza el acceso a los repositorios necesarios.

jules.google.com ↗
  • Sesión iniciada con Google
  • GitHub vinculado a Jules
  • Acceso autorizado a los repos necesarios
  • Jules ve repositorios de la organización Hypenosys

Los permisos exactos dependen de la configuración de GitHub, OAuth y la autorización de la organización — no hay automatismos garantizados aquí.

03Repo

Seleccionar el repositorio de prueba

Minecraft-Server-Docker

Este repositorio es un entorno seguro de prueba y puede estar vacío al inicio. Su única finalidad es comprobar que el pipeline Jules → GitHub → Pull Request funciona. Selecciónalo dentro de Jules antes de escribir el prompt.

04Tarea

Lanzar la primera tarea en Jules

Jules

Puedes usar el modo Interactive Plan si quieres revisar el enfoque antes de que Jules modifique archivos, o Start si quieres que Jules genere la solución directamente.

Prompt sugerido 
Quiero un Docker Compose para un servidor local de Minecraft para BoinaLand. El objetivo es tener una configuración inicial sencilla, reproducible y documentada. Incluye los archivos necesarios, una estructura clara del proyecto y un README con instrucciones para levantar el servidor en local.
Versión estricta 
Crea una configuración Docker Compose para un servidor local de Minecraft llamado BoinaLand. El repositorio puede estar vacío, así que genera la estructura mínima necesaria. Incluye docker-compose.yml, .env.example, carpetas persistentes si son necesarias y un README.md con instrucciones claras para iniciar, parar, configurar y actualizar el servidor. No uses secretos reales. Prioriza simplicidad, persistencia de datos y facilidad de mantenimiento.
05PR

Revisar cambios y Pull Request

PRReview

Si todo va bien, Jules debería generar cambios en el repositorio y permitir crear una Pull Request. Revisa los archivos antes de aprobar — la PR debe ser revisada antes de mergear.

  • docker-compose.yml existe
  • Hay volúmenes persistentes
  • Hay README.md
  • No hay secretos ni tokens reales
  • El nombre BoinaLand aparece donde corresponde
  • La configuración puede arrancar en local
  • La PR tiene descripción clara
Alternativa

Flujo interno desde Hypenosys

Además del flujo directo con jules.google.com, también puedes trabajar desde hypenosys.github.io, usando el Jules Panel y el Claude Chat del sistema.

  1. Entrar en hypenosys.github.io
  2. Abrir el Jules Panel
  3. Autenticarte con GitHub si el panel lo solicita
  4. Seleccionar la organización / repositorio correspondiente
  5. Usar el Chat Neural / Claude Chat para preparar o refinar el prompt
  6. Enviar la tarea a Jules desde el panel cuando el prompt esté validado
  7. Revisar el estado de la tarea, actividad, rama y Pull Request desde el panel

Claude Chat

Razona, prepara, refina y valida instrucciones antes de enviarlas.

Jules

Ejecuta tareas técnicas sobre los repositorios.

GitHub

Es la fuente de verdad para ramas, commits y Pull Requests.

Recomendaciones

Buenas prácticas

  • Empieza siempre en repositorios de prueba
  • No uses secretos reales
  • Pide README y documentación desde el primer prompt
  • Usa ramas separadas para cada tarea
  • Revisa la PR antes de mergear
  • Prefiere prompts concretos, con estructura esperada de archivos
  • Si Jules propone un plan, revísalo antes de aceptarlo
  • Si algo falla, copia el error exacto y documéntalo en el panel o en una issue
Troubleshooting

Errores comunes

Posibles causas:

  • No has aceptado la invitación a la organización
  • GitHub no ha autorizado a Jules para esa organización
  • Jules no tiene permisos sobre ese repo
  • El repositorio es privado y falta autorización

Posibles causas:

  • Falta permiso de escritura
  • La rama no se ha creado correctamente
  • Jules no tiene autorización suficiente
  • El repo está vacío y Jules necesita generar la estructura inicial

Posibles causas:

  • Falta autenticación con GitHub
  • Token caducado
  • No se ha refrescado la lista de repositorios
  • Todavía no perteneces a la organización correcta
